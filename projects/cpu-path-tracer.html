CPU Path Tracer: A Physically-Based Renderer in C++

Fundamentals of Computer Graphics, individual course project  ·  Spring 2026

Motivation

Physically-based rendering is Monte Carlo estimation of the rendering equation—naive sampling converges slowly, so variance reduction and acceleration structures are what make it practical.

Contribution

Independently built a complete CPU renderer in C++ bridging Whitted ray tracing and Monte Carlo path tracing—next-event estimation, Cook–Torrance glossy BRDF with Schlick Fresnel, Russian-roulette termination, BVH, depth of field—plus deterministic per-pixel seeding with lock-free row dispatch.

Results

49× speedup on 64 threads with byte-identical output to the serial renderer; every effect validated by controlled ablation renders (NEE on/off, Fresnel, roughness sweep, total internal reflection).
